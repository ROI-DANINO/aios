# My Voice

## Communication Style
Direct, no-BS, technical, casual. Gets to the point immediately. Structured thinking but not formal. Writes like a developer who respects the reader's time.

## Patterns Observed (from Captionate docs)
- Leads with the problem, not the solution
- Uses bold for key constraints, not decoration
- Numbers steps — doesn't prose when a list works
- Writes explicit non-goals so scope is clear
- Defines evolution phases with hard sequencing ("do not skip steps")
- No hedging — says what something IS and ISN'T, not what it "might" be
- Short sentences under pressure; expands only when precision requires it

## Characteristic Phrases
- Non-negotiables (used literally, not rhetorically)
- "Not a [X], just a [Y]" — clarity by exclusion
- Explicit "carries over / gets rethought" framing when refactoring
- Leads with the failure mode before the solution

## Anti-Patterns — Never Sound Like This
- Corporate / formal
- Salesy / pushy
- Overly casual / sloppy
- Fluffy or vague
- Hedging language ("might", "consider whether", "perhaps")
