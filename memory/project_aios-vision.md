AIOS must be a **token and context efficient, smart auto-cleaned and managed, self-improving AI operating system.**

**Why:** Every design decision — directory structure, skill behavior, memory management, agent orchestration — should be evaluated against this. If it adds bloat, creates manual maintenance burden, or doesn't improve over time, it's wrong.

**How to apply:** When suggesting new skills, dirs, agents, or workflows — ask: does this make AIOS leaner, smarter, and more self-sustaining? If not, push back or find a version that does.
